# Skill: Learning Item Granularity

Keep first-pass learning items concrete and usable.

## Required Output

Write `intermediate/first_pass/02_analysis/learning-item-granularity-analysis.md`.

## Standard

- Item granularity is by knowledge point.
- A normal item should take about 3–8 minutes.
- Ordinary sections should contain 5–10 items.
- Complex sections must not exceed 13 items.

## Too Broad

An item is too broad if it resembles a section title, bundles several definitions/formulas/processes, needs many pages of reading, or has a checkpoint question requiring a long essay.

## Too Narrow

An item is too narrow if it is only a symbol, variable name, phrase, or answerable by one word without independent reading.

An item is too narrow if it cannot support a meaningful scene-judgment MCQ — i.e., the item consists of a definition, isolated term, single condition, or rote fact with no application scenario, comparative dimension, or condition-dependent judgment space. Such items should be merged into the adjacent KP that uses or builds upon them.

## Repair

- Split too-broad items by definition, condition, process step, field group, formula-use condition, representation conversion, or failure condition.
- Merge too-narrow items into a neighboring reading task.
- For items too narrow for scene-judgment: merge into the consuming or neighboring KP. The merged content appears in the parent KP's quick_absorption interpretation section. Document the merge in kp-consolidation-analysis.md.
- If a section exceeds limits, prioritize and defer rather than flooding the first-pass pack.
