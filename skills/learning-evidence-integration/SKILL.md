# Skill: Learning Evidence Integration

Integrate evidence only after it has been analyzed. Evidence includes feedback, textbook body details, thinking questions, worked examples, exercise performance, and failed prior outputs.

For feedback, first state the stable knowledge revision requirement. A feedback map is invalid if it only assigns a location and does not state the knowledge requirement.

For thinking questions and worked examples, abstract them into models, conditions, boundaries, methods, or recognition cues. They may influence lesson details, examples, or practice, but must not become the lesson structure.

For practice-only or fluency issues, write `03_plans/problem-set-repair-plan.md` even when the final problem set is not requested. The plan can exist without generating a problem set.
