# Using Examples & Analogies

## 一句话定位

一个好的例子不是"让学生更容易理解"，而是"让学生建立从已知到未知的桥梁"。

## Related Theories

- **Worked Example Effect** (Sweller): Novices learn more from studying worked examples than from solving problems. Reduces cognitive load by directing attention to solution-relevant steps.
- **Analogical Reasoning** (Gentner): Learning by analogy transfers knowledge from a familiar domain to an unfamiliar one. The key is structural mapping — identifying what's the same at the relational level despite surface differences.
- **Expertise Reversal Effect** (see aha-moments/expertise-reversal.md): Worked examples help novices but can bore or constrain experts. As the learner advances, examples should be faded.

## Teaching Application

- In 速览, examples serve as recognition anchors — they help the student identify the concept when they encounter it in the source.
- The best examples share surface AND structure with the concept (literal analogies). The next best share structure only (abstract analogies). The worst share surface only (misleading analogies).
- When using analogies, explicitly state the mapping: "格林公式 : 闭曲线→区域 :: 高斯公式 : 闭曲面→体积."
- Avoid analogies that introduce more confusion than clarity. An analogy that requires explaining the analogy itself is worse than no analogy.

## Aha Moment

> Bad analogies teach the wrong things. "Memory is like a computer" sounds helpful but leads students to think memory is perfectly accurate and unlimited — which is the opposite of how human memory works.
